# V073 Dynamic Tactical Objectives Spec

## Goal

Attach small battle-local optional objectives to battlefield events so players have short tactical decisions during a fight without a giant quest system.

## Objective Types

- Hold a threatened site until the timer ends.
- Protect the Command Hall through a pressure window.
- Defeat an elite squad while the strike is active.
- Use or preserve the Retinue reinforcement opportunity.
- Use hero abilities during an Aether Surge window.

## Completion Rules

- Objectives are optional and battle-local.
- Event objectives do not block victory or defeat.
- Completion grants only small battle-local resources, mana, or after-action credit.
- Failures are readable but not punitive beyond the missed opportunity.
- Tutorial/no-reward routes do not spawn dynamic objectives.

## Farming Safety

- Dynamic objectives are not stored as repeatable campaign rewards.
- Event bonuses do not duplicate unique relics, first-clear rewards, Retinue recovery, or campaign optional-objective records.
- Replay battles can show events, but rewards remain battle-local and Results copy must not imply persistent farming.

## UI Scope

- HUD shows event title, objective, timer/progress, and one counterplay hint.
- Results shows encountered events, completed/failed objectives, and a short after-action note.
- Existing map secondary objectives remain separate and unchanged.

## Deferrals

- Multi-step event chains.
- Persistent tactical objective medals.
- New objective markers or art.
- Large branching rewards.

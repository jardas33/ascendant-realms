# v0.98 Stronghold UX Rescue Spec

Status: implemented presentation-only spec.

## Goals

Stronghold should communicate what can be upgraded now, what is locked, and what each upgrade does without a prose wall.

## Default Layout

The Stronghold panel now prioritizes:

- current tier;
- available upgrade count;
- locked upgrade count;
- purchased upgrade count;
- current campaign resources;
- upgrade cards with cost, prerequisite, benefit, and action.

## Upgrade Cards

Each upgrade card shows:

- name;
- tier and rank;
- concise description;
- cost;
- prerequisite status;
- benefit summary;
- Upgrade / Purchased action state.

Flavor and extended status copy are behind `More Details`.

## Rules Boundary

No Stronghold upgrade definitions, costs, prerequisites, effects, purchase rules, campaign resources, save fields, or launch modifiers changed.

## Deferrals

- Final Stronghold art.
- Animated upgrade tree.
- Building-level Stronghold view.

# v0.78 Visual Direction And AI Art Governance

Date: 2026-05-30

## Visual Target

A modern dark heroic-fantasy RTS/RPG with grounded Barrosan highland inspiration, readable tactical battlefields, strong silhouettes, tactile UI, dramatic lighting, fog, rain, wet terrain, granite, hearth fire, shrines, springs, mines, old roads, winter masks, and distinctive non-human races.

## What The Final Presentation Must Not Feel Like

- a 2000-era game,
- a 2010-era game,
- a generic browser game,
- a mobile-game reskin,
- a direct imitation of Warlords Battlecry,
- a direct imitation of Age of Empires,
- a direct imitation of Warcraft,
- a collage of inconsistent AI-generated assets.

## Visual Principle

Tactical readability first. Spectacle should improve clarity, atmosphere, and emotional impact rather than obscure gameplay.

## Core Pillars

- Strong race silhouettes before labels.
- Readable tactical scale for units, buildings, sites, roads, and hazards.
- Barrosan highland mood: granite, rain, fog, mines, shrines, hearths, wet roads.
- Distinct architecture per race.
- UI that feels tactile, dark, grounded, and command-ready.
- Effects that explain ability/state changes without covering targets.

## AI-Assisted Art Risks

- inconsistent characters,
- inconsistent race anatomy,
- inconsistent proportions,
- inconsistent buildings,
- inconsistent lighting,
- inconsistent perspective,
- inconsistent animation frames,
- unreadable silhouettes,
- visual drift between assets,
- style drift between factions,
- accidental resemblance to protected designs.

## Mitigation

- art-direction bible,
- race-specific silhouette sheets,
- architecture sheets,
- controlled prompt templates,
- reference boards using safe descriptive categories,
- approved style frames,
- asset review gates,
- source and license metadata,
- versioned intake,
- no runtime inclusion without review,
- screenshot QA,
- visual consistency checklists.

## Runtime Asset Rule

No AI-assisted image, generated frame, reference image, or candidate asset may enter runtime without source/license metadata, human style review, visual consistency review, and validation. v0.78 adds no art.

## Original-IP Rule

Prompting and style direction must describe function, mood, material, camera, scale, and gameplay readability. Do not use "in the style of" protected games, living artists, identifiable franchises, or copied assets.

## Future Production Checks

Before any visual runtime replacement:

1. silhouette sheet approved,
2. architecture sheet approved,
3. source/license metadata complete,
4. visual manifest updated,
5. screenshot QA plan defined,
6. package validation remains clean,
7. human review confirms the asset does not resemble protected expression.

# v0.138 02 HUD Style Frame

Status: copy-ready reference prompt only. Do not generate images inside this repository. Do not import generated images into runtime.

## Copy-Ready Prompt

Create one reference-only HUD style frame for an original fantasy RTS/RPG called JARDAS: Oath of the Barrosan Marches. Show a modern 2026 desktop fixed-camera RTS/RPG battle interface over a readable Salto battlefield. The HUD should support a central persistent hero, selected Worker or squad, resource row, compact objective guidance, minimap, command buttons, production progress, and Results-ready battle flow. Use practical Barrosan materials: dark iron, weathered timber, muted cloth, carved stone, clean ink labels, and restrained teal Lume highlights. Preserve battlefield visibility, especially the upper playfield, with compact panels and clear hierarchy. The UI should feel polished, tactical, and readable, not oversized marketing, not a generic developer dashboard, and not a generic mobile-game card stack.

Original IP only. Evoke a modern top-down RTS/RPG presentation with strong tactical readability and a central hero, while avoiding direct Warlords Battlecry imitation or any protected HUD, icon set, faction symbol, font treatment, layout, or game identity.

## Negative Prompt

No direct Warlords Battlecry UI imitation, no Warcraft UI, no Age of Empires UI, no copied command-card layout, no protected icons, no protected logos, no giant fantasy ornament covering the battlefield, no mobile-game card stacks, no MMO hotbar clutter, no developer-dashboard styling, no unreadable tiny type, no excessive baked-in text, no final UI kit, no runtime-ready assets.

## Generation Settings

- Aspect ratio: 16:9.
- Expected resolution: 1920 x 1080.
- Purpose: reference-only HUD style direction.

## Visual Questions

- Does the HUD leave the battlefield readable, especially the upper playfield?
- Can the selected hero, Worker or squad, objective, command buttons, resources, minimap, and progress feedback be found quickly?
- Does the interface feel like desktop RTS/RPG play rather than a dashboard or mobile UI?
- Are Barrosan and Lume cues present without becoming noisy?

## Rejection Triggers

- UI covers critical battlefield action or top-screen landmarks.
- Layout resembles a protected RTS HUD, icon set, or command grid too closely.
- The frame uses excessive baked-in text, unreadable labels, or over-decorated panels.
- The style reads as generic mobile fantasy or software dashboard.

## Originality And IP Boundary

The HUD must be an original interface direction for JARDAS. It can use broad RTS/RPG affordances such as minimap, resources, selected-unit panel, and command buttons, but must not copy protected UI layouts, symbols, fonts, panels, icons, or faction motifs.

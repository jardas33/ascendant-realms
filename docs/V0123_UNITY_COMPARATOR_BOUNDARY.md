# v0.123 Unity Comparator Boundary

Status: comparator boundary only. No Unity project is authorized by v0.123.

## When Unity Comparison Is Required

Run a Unity comparator only if one of these becomes true in a future explicit goal:

- Godot visual quality cannot support the desired fixed-camera RTS/RPG feel after a focused visual slice.
- Godot asset import becomes routine editor work or manual drag-and-drop.
- Godot fresh-checkout validation, Windows export, package, or CLI automation becomes unreliable.
- Godot Tier M or Tier L evidence regresses materially.
- Emmanuel reviews the Godot packet and asks for a direct comparator before approving further Godot expansion.

## What Unity Must Prove

A Unity comparator must use the same evidence contract as Godot:

- Same generated Salto fixture.
- Same Tier S/M/L workload scale.
- Same fixed-seed benchmarking posture.
- Same one-click validation goal.
- Same editor-optional routine workflow requirement.
- Same manifest-driven content import requirement.
- Same stable-ID rejection posture.
- Same read-only save-fixture posture.
- Same Windows export and package requirement.
- Same no-art posture.
- Same no final engine decision boundary.

## Required Unity Outputs If Authorized Later

- Repository-local Unity project files only, if a future prompt explicitly authorizes a Unity project.
- Scripted setup and validation from a fresh checkout.
- CLI or one-click build/export/package.
- Deterministic reports for fixture import, stable IDs, saves, S/M/L benchmark, package, and scorecard.
- Explicit list of unavoidable manual editor steps, preferably empty.

## Blocked In v0.123

- Creating a Unity project.
- Installing Unity.
- Downloading Unity assets or packages.
- Creating prefabs, scenes, or import settings.
- Importing art.
- Comparing final engine choice.
- Starting a Unity port.

## Comparator Decision Rule

Unity should be treated as a comparator only when it can prove better visual potential without losing the AI-first, editor-optional, deterministic automation contract. If Unity requires routine manual editor work, it fails the core Emmanuel requirement even if the visuals are promising.

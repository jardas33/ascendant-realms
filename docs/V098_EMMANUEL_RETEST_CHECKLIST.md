# v0.98 Emmanuel Retest Checklist

Use the final clean package named `ascendant-realms-private-playtest-<final commit>` with no `-dirty` suffix.

## Ten-Minute Retest

1. Launch the package with `START_GAME_WINDOWS.bat`.
2. Open Hero Inventory from the main menu or campaign map.
3. Check that Hero Overview explains level, XP, class, origin, stats, equipment, relic, skill points, Retinue, and inventory without feeling like a text dump.
4. Check Skills: available / purchased / locked states, cost, requirement, and effect should be readable quickly.
5. Check Equipment and Inventory: equipped versus stored should be obvious; relic effects should say they are active only when equipped.
6. Continue Campaign and open Hero tab: Retinue Ready / Deployed / Recovering / Reserve status should be clear.
7. Open Stronghold tab: current tier, available upgrades, locked upgrades, costs, benefits, and action state should be clear.
8. Complete or launch a test battle to Results: compact summary should still lead; progression summary should clarify XP, rewards, relic, Retinue, and Stronghold/campaign consequences.
9. Open Results full details: data should remain present but no longer dominate the first read.

## Questions

- Is the Hero screen now understandable within a few seconds?
- Is the Skills area readable without needing a guide?
- Is equipped versus stored gear clear?
- Is the Retinue state clear enough before battle?
- Is the Stronghold upgrade action clear?
- Does Results make the next meta-progression step obvious?

## Boundary

v0.98 changes no saves, IDs, rewards, XP, hero rules, relic stats, equipment rules, Retinue rules, Stronghold rules, campaign progression, gameplay balance, maps, factions, art, or desktop work.

# v0.12.6 Route Assignment Plan

Date: 2026-05-18

Use this plan to assign manual playtest routes without overloading testers.

## Route Set

| Route | What It Reveals |
| --- | --- |
| Baseline Cautious | Normal first-player clarity, objective order, fair losses, and pressure noticeability without optimization. |
| No-Retinue | Whether the campaign remains fair without carried veteran power. |
| One-Veteran | Whether one meaningful carryover advantage feels earned rather than dominant. |
| Mixed-Veterans | Normal experienced-player play with some carryover but not maximum stacking. |
| Retinue + Training Yard II | Whether the strongest known profile trivializes Ashen/Cinderfen or still feels earned. |
| Greedy Economy | Whether resource/value play is clear, fair, and conversion-limited rather than secretly underpowered. |
| Fast Army | Whether quick army timing rewards decisive play or invalidates slower profiles. |

## For 1 Tester

Assign:

1. Baseline Cautious

Reason: one tester should first answer whether the current game is understandable and fair without special optimization.

## For 2 Testers

Assign:

1. Baseline Cautious
2. Fast Army

Reason: compare normal cautious play against decisive speed without needing prepared veteran-state setup.

## For 3 Testers

Assign:

1. Baseline Cautious
2. Greedy Economy
3. Retinue + Training Yard II

Reason: cover the normal route, the risky economy route, and the strongest known route.

If Retinue + Training Yard II cannot be set up cleanly, replace it with Fast Army and mark the strong-route test as not yet covered.

## For 5 Testers

Assign:

1. Baseline Cautious
2. No-Retinue
3. One-Veteran
4. Greedy Economy
5. Fast Army

Reason: this gives a fair spread across no carryover, light carryover, economy pressure, and speed pressure.

## For 7 Testers

Assign all major routes:

1. Baseline Cautious
2. No-Retinue
3. One-Veteran
4. Mixed-Veterans
5. Retinue + Training Yard II
6. Greedy Economy
7. Fast Army

Reason: this covers the full v0.12.x watchpoint set while keeping each tester focused.

## What Each Tester Should Play

Preferred node order:

1. Tutorial / Proving Grounds or Border Village as a warm-up.
2. Ashen Outpost if available.
3. Cinderfen Crossing if available.
4. Cinderfen Watch if available.
5. Results screen and campaign return flow after one win or loss.

If a route needs a prepared save or profile, do not ask the tester to edit files. Provide the save/profile or mark the route as not tested.

## Pass / Fail Concern Triggers

Record a concern when:

- the tester cannot tell what to do first
- command feedback repeatedly feels missing or delayed
- pressure warnings are shown but missed during combat
- defeat feels unfair after the tester understood the objective
- a route wins while ignoring objectives and pressure
- a route loses repeatedly despite clear play and reasonable reactions
- results do not explain what happened or what to try next

Do not treat a single unlucky run, a missing prepared save, or a pure art complaint as a route verdict.

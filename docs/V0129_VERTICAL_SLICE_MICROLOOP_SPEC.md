# v0.129 Vertical Slice Microloop Spec

v0.129 adds one bounded Godot-only gameplay proof for the player-facing Salto slice. It keeps the browser prototype authoritative and does not approve a full port, final Godot choice, save migration, stable-ID migration, art import, campaign expansion, broad economy, broad building tree, broad recruitment, or full hero progression.

## Microloop

1. Aster can be selected, moved toward the mine/quarry posture, and use one placeholder Rally ability.
2. Aster converts the West Stone Cut mine site.
3. The Worker can be assigned to the converted mine.
4. Worker presence modestly boosts Stone/Iron production inside the runtime-only proof loop.
5. The Worker restores the existing Barracks placeholder through a construction-progress state.
6. The Barracks queues and trains one Militia with a clear resource spend.
7. A single Ashen pressure wave is triggered and defeated.
8. The Lume route is restored and the slice reaches Results.

## Boundaries

- No browser runtime, browser saves, localStorage, stable IDs, imported art, generated image assets, full economy, full build tree, broad recruitment, campaign, or final engine selection changed.
- `linked_ward` remains exactly `0.92`.
- The loop is reset by workload tier and remains deterministic from repository files and scripts.
- Routine Godot editor work is not required.

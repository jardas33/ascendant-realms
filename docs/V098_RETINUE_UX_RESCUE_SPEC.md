# v0.98 Retinue UX Rescue Spec

Status: implemented presentation-only spec.

## Goals

The Retinue Camp should make survivor continuity readable at a glance without becoming a permanent army-management game.

## Visible Summary

The Retinue panel now shows:

- roster count and cap;
- deployed count and cap;
- ready reserve count;
- recovering count;
- Training Yard II deployment effect;
- reinforcement eligibility summary.

## Unit Cards

Each saved veteran card surfaces:

- Ready / Deployed / Recovering state;
- reserve/deployment posture;
- veteran rank;
- role;
- XP progress and kills;
- reinforcement eligibility or recovery block reason;
- deployment/reserve action;
- dismiss action.

Member details such as unit type, rank bonus, flavor, battles survived, and missions deployed are behind `Member Details`.

## Rules Boundary

Retinue capacity, deployment selection, recovery timing, reinforcement eligibility, loss/removal behavior, Tutorial protection, and save normalization are unchanged.

## Deferrals

- Persistent roster sorting/filtering.
- Large roster UI.
- Retinue equipment or army inventory.

# v0.116 Emmanuel Architecture Review Packet

## Summary

v0.116 prepares an architecture-review packet and engine-neutral Salto fixture for future desktop spike discussion. It does not select an engine, start a port, create a wrapper, or integrate runtime art.

The current browser evidence remains RED. v0.115 reports representative Tier M rows around 2.4 to 2.5 FPS with p95 frame time in the 533.3 to 633.3 ms range. That blocks browser runtime visual expansion and motivates a measured architecture review.

## What Emmanuel Needs To Review

- Whether the recommended future spike order is sensible: Godot first, Unity second, Electron as a control, Unreal deferred.
- Whether the AI-first workflow requirements are strong enough.
- Whether the visual ambition is aligned with the desired game identity.
- Whether any future engine spike should be authorized after v0.116 is closed.

## AI-First Workflow Requirement

Future engine workflow should be designed for Codex and automation first. Emmanuel should not need to operate a game engine manually for routine development.

Emmanuel can help with:

- Installing required software.
- Running one-click or simple CLI scripts.
- Launching packaged builds.
- Recording short playtests.
- Approving visual direction.
- Answering creative and product questions.

The future spike must prove Codex can:

- Create project setup from repository files.
- Create or modify the representative Salto scene with scripts or text-editable files wherever practical.
- Import content from manifests and validate it.
- Register routine assets without manual editor drag-and-drop.
- Run builds, exports, tests, benchmarks, and Windows packaging through scripts.
- Produce scorecard evidence.
- Identify and minimize unavoidable manual editor steps.

## Visual Ambition

The target should evoke the spirit of a super-cool 2026 evolution of Warlords Battlecry while staying fully original: original factions, lore, assets, UI, mechanics, worldbuilding, and visual identity.

The future benchmark should assess:

- Modern top-down RTS/RPG presentation.
- Polished 2D versus 2.5D fixed-camera approaches.
- Strong Barrosan and Ashen faction silhouettes.
- Atmospheric Salto terrain.
- Modern lighting and VFX.
- Central persistent hero readability.
- Tactical command clarity.
- A polished game UI that does not resemble a mobile game or developer dashboard.

## Recommended Candidate Review

| Candidate | Recommended Role |
| --- | --- |
| Godot 4 fixed-camera 2D/2.5D | First future spike candidate because it may support scriptable, text-friendly, AI-operable workflows with strong 2D/2.5D presentation. |
| Unity 2D/URP fixed-camera | Second future spike candidate because it has strong visual tooling but must prove reproducible editor-optional automation. |
| Electron/browser wrapper | Packaging control only; current browser battle-loop evidence is RED, so a wrapper is not a cure. |
| Unreal | Deferred because it is likely too heavyweight and editor-centric for the current fixed-camera slice. |

- Can Codex generate or update the Salto representative scene?
- Can content import stay manifest-driven and stable-ID validated?

- Can benchmark data fill `docs/V0116_ENGINE_SPIKE_SCORECARD_TEMPLATE.json`?
- Can Emmanuel avoid routine editor operation?

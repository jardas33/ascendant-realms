# v0.120 Zero-Editor Automation Contract

Emmanuel should not need to operate the Godot editor for routine v0.120 workflow validation.

## Codex-Owned Work

Codex and repository automation own:

- fixture export and validation;
- stable-ID validation;
- read-only save-fixture validation;
- scene generation;
- headless Godot project validation;
- headless tests;
- 2D and 2.5D benchmark evidence;
- Windows export;
- Windows package ZIP generation;
- package hash validation;
- temp checkout cleanup;
- CI-style manual workflow posture.

## Emmanuel-Owned Work

Emmanuel may:

- install or approve official local tools when explicitly needed;
- double-click the one-click scripts;
- launch the packaged build;
- record short playtests;
- approve visual direction;
- answer creative questions.

## Not Routine

The following are not routine v0.120 actions:

- opening the Godot editor to assemble scenes;
- dragging files into the editor;
- manually registering assets;
- importing artwork;
- changing project settings through editor-only state;
- selecting Godot as the final engine;
- starting a full port.

## Acceptance

The future spike remains acceptable only if Codex can create, modify, validate, build, export, package, and report the representative Salto slice from repository files and simple scripts. Any unavoidable manual editor step must be recorded explicitly before it is accepted.

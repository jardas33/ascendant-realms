# v0.8.2 Manifest Metadata Refinement

Date: 2026-05-10  
Scope: conservative source/license metadata refinement based on `docs/V082_ASSET_SOURCE_LICENSE_AUDIT.md`. This phase does not move, delete, rename, generate, download, replace, or approve assets.

## What Changed

The visual asset manifest schema now includes two source-review fields:

- `reviewStatus`
- `sourceReviewNotes`

These fields are metadata only. They do not alter runtime asset loading, rendering, CSS variables, gameplay, saves, campaign progression, screenshot QA, or asset files.

## Review Status Values

Allowed review statuses:

- `approved-for-prototype`
- `approved-for-production`
- `needs-source-proof`
- `generated-review-needed`
- `reference-only`
- `do-not-ship`
- `deprecated`

The current v0.8.2 manifest uses only:

| reviewStatus | Count | Meaning |
| --- | ---: | --- |
| `needs-source-proof` | 87 | Current file-backed image assets still lack explicit author/source/license evidence. |
| `approved-for-prototype` | 2 | Owned local metadata or original procedural renderer metadata is acceptable for prototype use, but not production art. |

No current asset is marked `approved-for-production`.

## Source Review Notes

`sourceReviewNotes` separates legal/source review from the broader visual-quality `notes` field.

Default note for current unknown-license image assets:

```text
No explicit author, generation method, or license proof is attached yet. Keep prototype-only until source/license evidence is reviewed.
```

Owned local metadata override:

- `asset_prompt_book`: owned local prompt/reference metadata generated by the repo pipeline; not runtime art.

Original procedural runtime metadata override:

- `procedural_battle_terrain`: original project renderer code, source-safe for prototype use, but visually placeholder and not production art.

## What Did Not Change

- No image asset changed.
- No image asset was promoted to production-safe.
- No asset was marked `final`.
- No unknown-license runtime image became `allowedInProduction`.
- No gameplay, campaign, save, pressure, map, unit, faction, worker/construction, renderer, or UI behavior changed.

## Manifest Posture After Refinement

The manifest now says the quiet part out loud:

- Most current images are prototype-useful but still need source proof.
- The prompt book and procedural terrain metadata are owned/prototype-safe, but they are not production art.
- The `public/assets/final/` folder name remains a processing/runtime-priority label, not a legal or visual-final claim.
- Future visual replacement should start with runtime-visible unknown-license assets and critical/high replacement-priority entries.

## Validation Update

The validator now checks that `reviewStatus` is one of the allowed values. Phase 4 should harden validation further around combinations such as:

- `reviewStatus: "approved-for-production"`
- `needsReview: true`
- `replacementPriority: "critical"`
- `currentStatus: "candidate"`
- `licenseStatus: "reference-only"` or `do-not-ship`

The goal is to preserve honest unknowns while preventing accidental production claims.

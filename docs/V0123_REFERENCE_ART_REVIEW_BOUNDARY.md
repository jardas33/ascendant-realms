# v0.123 Reference Art Review Boundary

Status: reference-art boundary only. v0.123 does not generate images and does not import runtime art.

## Allowed

- GPT image generation outside runtime after Emmanuel approves the prompt text.
- Human review of generated candidates.
- Contact sheets in ignored artifact folders.
- Prompt revision.
- Style approval as reference-only.
- Protected-IP review.
- Source and license posture review.
- Notes that guide a future visual slice.

## Blocked

- Runtime import.
- Asset registration as runtime-integrated.
- Mass generation.
- Full roster generation.
- Complete UI kit generation.
- Final terrain set generation.
- Runtime sprites.
- Runtime buildings.
- Icons for runtime use.
- Engine-specific art import settings.
- Final engine commitment.
- Any claim that reference approval is loadable runtime approval.

## Required Review Language

Every v0.123 prompt must preserve these statements:

- The image is reference-only.
- Runtime integration is explicitly forbidden.
- Original IP is required.
- Protected lookalikes are rejected.
- Human review is required before any future use.

## Future Runtime Gate

Runtime art requires a separate future milestone with:

- source/license review;
- protected-IP review;
- Emmanuel approval;
- manifest-driven asset registration;
- runtime slot validation;
- visual QA;
- package evidence;
- explicit confirmation that the asset is no longer reference-only.

No such gate is part of v0.123.

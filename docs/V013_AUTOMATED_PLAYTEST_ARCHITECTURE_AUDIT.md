## Current Metrics

Already tracked:

- battle result: victory / defeat / timeout
- node, map, difficulty, AI personality
- script and Stronghold profile
- starting resources and starting units
- first capture timing
- Barracks placed/completed timing
- first unit trained timing
- first enemy warning/contact timing
- enemy pressure plan id
- triggered pressure stages
- pressure warnings shown
- losses after pressure
- reinforcement application flag
- first-wave survival
- units trained/lost
- buildings built
- upgrades researched
- enemy waves survived
- enemy hero/rival telemetry
- objective completion ids
- battle duration
- resources floated and peak resources floated
- hero level and XP gained
- final army size
- reward result
- command log
- structural notes

## Missing Or Weak Metrics

Unavailable today:

- final hero HP
- explicit hero death/survival state
- base damage / base HP
- exact objective timing/order beyond command-log inference
- human noticeability
- human confusion
- visual readability
- fun
- whether a warning was actually seen by a human

Derived but imperfect:

- pressure reaction window can be approximated from first pressure time to first enemy contact when both exist.
- likely failure reason can be inferred from result, first-wave survival, pressure losses, resources, and objective completion.
- route/watchpoint markers can be inferred from scripts, Stronghold profile ids, retinue, and upgrades.

## Safe Additions

Safe v0.13 additions:

- typed automated scenario-lab profiles layered over existing telemetry
- derived metric summaries
- watchpoint classifier rules
- Markdown/JSON lab outputs
- package scripts for lab/profile/watchpoint reports
- tests for profile metadata, classifier behavior, report sections, and no human-feedback claims

These additions do not require gameplay runtime changes.

## Deferred

Defer until real human feedback or a separately scoped implementation goal:

- any numeric tuning
- new maps/factions/units
- new AI construction/economy systems
- save migration
- runtime art or visual overhaul
- browser/UI gameplay changes
- hero HP/base-damage telemetry if it requires runtime battle model redesign
- interpreting automated warning telemetry as proof of human noticeability

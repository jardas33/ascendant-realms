# v0.13 Telemetry Metrics Spec

Date: 2026-05-18

Purpose: define which automated scenario-lab metrics are available, derived, or unavailable.

## Output Files

Generated by `npm run playtest:lab`:

- `PLAYTEST_SCENARIO_LAB.json`
- `PLAYTEST_SCENARIO_LAB.md`
- `PLAYTEST_WATCHPOINT_SUMMARY.md`

Generated by `npm run playtest:profiles`:

- `PLAYTEST_SCENARIO_PROFILES.json`
- `PLAYTEST_SCENARIO_PROFILES.md`

## Metric Availability

| Metric | Status | Source / note |
| --- | --- | --- |
| Win/loss/timeout | available | `battleResult`. |
| Failure reason | derived | Inferred from result, first-wave survival, pressure losses, resource surplus, losses, and objective completion. |
| Battle node | available | `nodeId`, `nodeName`, `mapId`. |
| Profile | derived | Scenario-lab profile definitions map scripts and Stronghold rows into route profiles. |
| Clear time | available | `battleDurationSeconds`. |
| Hero survival | unavailable | Current telemetry has hero level/XP, not final HP or death. |
| Army survival | available | `finalArmySize`, `unitsLost`, `unitsTrained`. |
| Resource surplus/shortage | available/derived | `resourcesFloated`, `peakResourcesFloated`, and structural notes. |
| Aether state | available | Final/peak Aether from resource bags. |
| Objective completion | available | `objectiveCompletion` ids. Exact timings are not first-class telemetry yet. |
| Pressure warning count | available | `pressureWarningsShown`. |
| Pressure reaction window | derived | `timeFirstEnemyContact - firstPressureTime` when both exist. |
| Base damage/base pressure | unavailable | Current simulator records base collapse notes, not base HP damage. |
| Retinue used | derived | Source telemetry retinue list for each matched run. |
| Training Yard II active | derived | Source telemetry purchased upgrade ids for each matched run. |
| Greedy Economy marker | derived | `playerScript === greedy_economy`. |
| Fast Army marker | derived | `playerScript === fast_army`. |
| Route verdict | derived | Scenario-lab classifier over profile summaries. |
| Confidence level | derived | Based on matched run count and signal strength. |
| Human noticeability | unavailable | Requires real human testing. |

## Failure Reason Categories

| Failure reason | Meaning |
| --- | --- |
| `none` | Victory. |
| `early_pressure_defeat` | Defeat before surviving first wave. |
| `pressure_attrition_defeat` | Defeat with losses after pressure triggers. |
| `failed_assault_defeat` | Defeat after a failed final assault or base collapse that is not an opening failure. |
| `conversion_timeout` | Timeout with high resource surplus and limited conversion into army. |
| `attrition_timeout` | Timeout with substantial losses or pressure attrition. |
| `clear_speed_timeout` | Timeout without completing the win condition. |
| `unknown_timeout` | Timeout not cleanly classified by current telemetry. |

## Do Not Fake

Do not infer:

- final hero HP
- actual human attention
- visual readability
- player confusion
- fun
- base HP damage

If these are needed, record them as unavailable or request real human/browser evidence.

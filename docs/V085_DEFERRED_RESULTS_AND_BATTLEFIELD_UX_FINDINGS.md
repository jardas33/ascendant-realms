# v0.85 Deferred Results And Battlefield UX Findings

v0.85 intentionally fixes only contextual Lume visibility and private-demo Results readability. The following findings are recorded without broad implementation.

## Deferred Battlefield UX

- Right-side command-panel density remains a broader HUD problem.
- Routine notification noise across pressure, events, objectives, and commands needs a later pass.
- Fog-of-war hard rectangles need a visual-design pass.
- Minimap scale and clarity need a separate review.
- Entity-label overlap during dense combat needs a broader label-priority pass.
- Selected-unit readability can still be improved beyond Lume site selection.
- The campaign shell likely needs a second polish pass after more manual feedback.

## Deferred Results UX

- Normal Results screens could benefit from progressive disclosure, but this checkpoint only applies it to private-demo Results.
- Battle rewards, Hero XP, Retinue, veterans, doctrine, tactical plan, Lume, events, and hero stats could use a broader hierarchy pass.
- Accessibility settings for battlefield overlays should be persistent only in a later settings milestone.

## Reason

The current checkpoint must not start v0.86, a broad battlefield-shell redesign, a broad Results redesign, or a persistent overlay-settings system.

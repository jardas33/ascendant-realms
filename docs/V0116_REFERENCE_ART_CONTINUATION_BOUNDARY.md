# v0.116 Reference Art Continuation Boundary

## Status

Reference-only. v0.116 does not generate art, import art, approve runtime art paths, or integrate art into the browser runtime or a desktop engine.

## Current Boundary

The v0.107 Salto slice remains the active art-composition reference. All planned assets remain placeholders or review candidates until a future explicit runtime art integration milestone.

Runtime fallback ownership remains with current placeholder/procedural systems. No asset is promoted to runtime use by v0.116.

## What Future Spikes May Evaluate

A future engine spike may evaluate whether a candidate can support the intended visual ambition:

- Original 2026-style RTS/RPG presentation.
- Polished top-down 2D or fixed-camera 2.5D composition.
- Barrosan and Ashen silhouettes that read at RTS scale.
- Atmospheric Salto terrain with road, ford, quarry, shrine, ruin, and readable buildable space.
- Modern lighting and VFX that improve mood without harming tactical readability.
- Central persistent hero presence.
- HUD, minimap, campaign, and Results frames that feel like a polished game, not a mobile UI or dashboard.

## AI-First Art Pipeline Requirement

Routine asset registration must be manifest-driven and validated automatically. Emmanuel should not need to drag files into an editor for ordinary iteration.

Future spikes should prove:

- Asset manifests can be generated or edited by Codex.
- Imported assets are validated by scripts.
- Runtime slot ownership is explicit.
- Missing or unapproved art falls back safely.
- Any unavoidable editor import step is documented and minimized.

## Protected-IP Boundary

The visual ambition may evoke the spirit of Warlords Battlecry as a genre reference, but Ascendant Realms must remain original in IP, factions, lore, assets, UI, mechanics, worldbuilding, names, and visual identity.

## Prohibited In v0.116

- Generated images.
- New asset binaries.
- Runtime sprite, terrain, building, UI, VFX, or icon imports.
- Changes to `public/` art paths.
- Changes to runtime-art slot assignment.
- Package metadata changes for art.
- Engine-specific art import settings.

## Required Future Gate Before Runtime Art

Runtime art requires a future explicit milestone with source/license review, protected-IP review, Emmanuel approval, runtime slot validation, packaging evidence, and visual QA.
